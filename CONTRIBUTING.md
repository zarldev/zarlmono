# Contributing

Thanks for your interest. A few things to read before opening a PR.

## Layout

This is a Go multi-module workspace:

- [`zkit/`](zkit/) — the shared substrate: agent runner, pursue
  (verified completion), guardrails, compaction, the tool system,
  LLM providers, and infrastructure packages. This is the library
  surface; changes here affect every application.
- [`zarlcode/`](zarlcode/) — the coding-agent TUI (own Go module).
- [`swebench-eval/`](swebench-eval/) — the SWE-bench evaluation driver
  (own Go module); drives the same loop as the TUI via
  `zkit/agent/coderunner` so the two cannot drift.
- [`examples/`](examples/) — six runnable examples (own module), each
  demonstrating one pattern; most run without an LLM via `-scripted`.
- [`zarlai/`](zarlai/) — the assistant application (own module, own
  [CONTRIBUTING](zarlai/CONTRIBUTING.md)).

[zarldev.github.io/zarlmono](https://zarldev.github.io/zarlmono) is the
user-facing documentation (source in [`site/`](site/) — Astro
Starlight; `cd site && npm run build` must pass when touching it).

## Workflow

- Branch off `main` — `feat/...`, `fix/...`, `chore/...`, `docs/...`
  per the existing convention. Keep PRs focused: one logical change.
- Verify locally before pushing — CI runs exactly this per module:

  ```bash
  go tool task check   # build + vet + test for every CI module
  go tool task lint    # golangci-lint, root .golangci.yaml config
  go tool task race    # zkit race suite
  ```

## Style

Go style lives in [`AGENTS.md`](AGENTS.md); the load-bearing rules:

- **Errors tell a story** — wrap with `fmt.Errorf("context: %w", err)`
  at every failure point; log once at boundaries; never "failed to" /
  "unable to" / "could not" prefixes.
- **Constructors return concrete types** — `NewX` returns `*X`; the
  interface satisfaction happens at the call site.
- **Small, emergent interfaces** — defined on the consumer side.
- **Every exported symbol is documented**, including interface-impl
  methods, whose docs state the implementation-specific behavior
  (caps, side effects, refusals) — never generic "implements X".
- **Fakes over mocks** in tests; black-box `package x_test` files,
  table-driven with `t.Run`; `*_internal_test.go` for white-box.
- **No fire-and-forget goroutines** — every goroutine has an owner and
  a shutdown path.

## Gotchas

- **Don't `go get -u` blindly**: `github.com/invopop/jsonschema` MUST
  stay at v0.13.0 in every module (v0.14 breaks anthropic-sdk-go).
  Re-pin after any upgrade sweep.
- Enums are generated by [goenums](https://github.com/zarldev/goenums)
  — edit the `*_enum.go`/`enums.go` source and `go generate`, never
  the `*_enums.go` output.
- Provider registration uses `init()` side effects in
  `zkit/ai/llm/...` deliberately; don't remove "unused" imports there.

## Security

The tools in this repository execute processes and mutate files by
design — `zkit` is shared infrastructure, not a sandbox. Report
vulnerabilities privately to a maintainer.
